Cascading Style Sheets, referred to as CSS, is a design language that allows to format the layout of the webpage. Text styles, table sizes, picture formatting and other aspects of Web pages can be defined by using this particular design language.

CASCADING STYLE SHEETS

Reasons of using CSS can be; HTML allows to add the contents to the page but CSS allows the user to style the page to his/her preferences, allows to save time as the user has to write CSS once and can be reused the same CSS sheet in multiple HTML pages, allows to create compatibility for different size of the screen displays and also creates easy maintenance as only one CSS sheet needed to be changed and it would update all elements in all the web pages automatically.

In today’ generation, technology has grown rapidly, where everyone gets updated through smartphones, rather than watching a television or reading a newspaper. This suggest, users does most of their activities mainly through their phones, such as; online banking, online shopping, reading news on internet, etc. As a result, web developers play a major role in these websites. Since, not only coding websites for desktop computers but also for mobile phones. Therefore, these websites need to be mobile responsive, in other words, needs to change its resolution according to the user's device. This can be achived by using 'media queries' that allows to be responsive to a particular device.

Going back to where ECMAScript was created, in 1995 JavaScript was created by Brendan Eich at Netscape. Initially, founder of Netscape, Marc Andreessen came up with the names called Mocha and later on it changed to LiveScript. In 1995, Netscape renamed LiveScript to JavaScript. Later on, Netscape introduced JavaScript to the Ecma standards organization, and it was renamed to ECMA-262, known as ECMAScript. This is being used widely in web pages as this allows to affect how web pages behave for the user.

ECMA Script

ECMAScript is usually used for client-side scripting on web pages. JavaScript, ActionScript and Jscript are examples of implementations of ECMAScript. Introducing ECMAScript has led many programmers to do their projects quickly and efficiently. Having ECMAScript classes, provides simpler and understandable syntax to create objects. It also consists ‘Promises’, allowing to understand the structure of the code. Having arrow functions allows an implicit return when there is no body block.
